Tezov Bank Application

L'application "Tezov Bank" (dénommée cette "App") présente une application de démonstration. Toutes les données présentées sont fictives. Cette App ne vend rien et ne collecte aucune donnée. Elle est totalement gratuite d'utilisation et est la propriété de Tezov.


A savoir :

  • Tous les identifiants et mots de passe sont acceptés. Il suffit de mettre 4 chiffres.
  • Toutes les pages ne sont pas implémentées. Quand il n'y a pas de contenu, un snackbar vous en informe.

Vous pouvez parcourir la suite de ce document pour avoir des informations supplémentaires.

** Démarrer l'application **

En savoir plus

Cette application est réalisée majoritairement en Kotlin. L'interface visuelle est entièrement réalisée avec Compose.

  • Architecture : 3 modules (core - 100% Kotlin, core - Android UI, application - UI)
  • Injection de dépendance avec Dagger multi-module (State et Action fournis à l'UI)
  • Séparation de l'UI avec injection du State (données + états de la vue) et Action (gestion des actions de l'utilisateur)
  • Navigation en seule source de vérité (coordinator pattern)
  • Chaque module a un module de test associé.
  • La totalité du contenu affiché par l'UI est reçue par le viewModel (Sauf bottom navigation, top app bar et icône/image). La vue se construit en fonction des données reçues du viewModel

En cours d'ajout :

  • Un module qui contiendra le viewModel
  • Un module qui contiendra la logique serveur
  • Un module qui contiendra les fonctionnalités propres à la plateforme (réseaux, persistance de données, caméra, etc...)

Le tout organisé avec une logique de clean architecture.

UI

Liste des composants UI que l'on trouve dans cette application :

  • Bottom navigation bar
  • Top navigation bar
  • Dialog
  • Bottom sheet
  • Snackbar
  • Tablayout
  • Pager (plein écran et carousel)
  • Collapsible animated header (Compose animation + Motion Layout + Custom Layout)
  • Drop down menu
  • Vue personnalisée cf. clavier de connexion

Le theming est intégralement réalisé en Kotlin pour Compose. Un thème général est fourni à tous les composants, peu importe le niveau de hiérarchie. Chaque composant peut personnaliser son propre style. L'architecture du thème est fournie par le module core - Android UI qui complète le materialTheme et apporte une notion d'états (default, success, error, alert, ...) d'un composant affectant le jeu de couleurs/graisse/dimension, etc.
